# Настройка переменных окружения для фронтенда

## Проблема: Фронтенд не может подключиться к бекенду

Если фронтенд потерял связь с бекендом, скорее всего не установлена переменная окружения `NEXT_PUBLIC_API_URL` в Vercel.

## Решение

### 1. Установите переменную окружения в Vercel

1. Откройте ваш проект в Vercel
2. Перейдите в **Settings** → **Environment Variables**
3. Добавьте новую переменную:
   - **Name**: `NEXT_PUBLIC_API_URL`
   - **Value**: URL вашего бекенда на Railway (например, `https://your-backend.railway.app/api`)
   - **Environment**: Выберите `Production`, `Preview`, и `Development` (или только нужные)

### 2. Пересоберите проект

После добавления переменной окружения:
1. Перейдите в **Deployments**
2. Нажмите на три точки рядом с последним деплоем
3. Выберите **Redeploy**

Или просто сделайте новый коммит и пуш в репозиторий - Vercel автоматически пересоберет проект.

## Проверка

После пересборки проверьте в консоли браузера (F12):
- Если видите ошибку `⚠️ ВНИМАНИЕ: NEXT_PUBLIC_API_URL не установлена`, значит переменная все еще не установлена
- Если ошибок нет, но запросы не проходят, проверьте CORS настройки на бекенде

## Важно

- Переменные окружения, начинающиеся с `NEXT_PUBLIC_`, доступны на клиенте
- Они должны быть установлены **до** сборки проекта
- После изменения переменных окружения нужно пересобрать проект

## Пример значения

Если ваш бекенд развернут на Railway и имеет URL `https://mariko-backend-production.up.railway.app`, то:

```
NEXT_PUBLIC_API_URL=https://mariko-backend-production.up.railway.app/api
```

Обратите внимание: URL должен заканчиваться на `/api` или функция автоматически добавит этот суффикс.
